## プロジェクト概要

### 目的

生駒祭 2026 の整理券発券システム（Supabase）にあるデータを取得し、
**配布率・待ち時間・残り枠**を関係者が即時に確認できるようにする。
本システムは**表示専用**とし、新規テーブル作成や編集処理は行わない。

### スケジュール

- 開発開始：2026/02/09
- リリース予定：未定

## 機能要件

### 1. 順次案内制（sequential）

**概要：**
時間枠を持たず、番号順に呼び出す方式。現在の呼び出し番号と最後尾番号から待ち時間を推定する。

**詳細仕様：**

- 対象テーブル: `events`, `event_dates`, `tickets`, `call_status`
- 企画の基本情報（企画名、場所）を表示
- 現在の呼び出し番号を表示（`call_status.current_call_number`）
- 最後尾番号は `event_dates.next_ticket_number - 1` として表示
- 待ち時間は以下の計算式で推定表示する
  - $\text{待ち時間(分)}=(\text{最後尾番号}-\text{現在の呼び出し番号})\times\text{1番号あたり待ち時間}$
- 1番号あたり待ち時間は `events.estimated_wait_minutes` を使用
- 発券ステータス（`event_dates.status`）により表示を切替

### 2. 時間枠定員制（time_slot）

**概要：**
時間枠ごとに定員がある方式。時間枠単位の残り枠を表示する。

**詳細仕様：**

- 対象テーブル: `events`, `event_dates`, `time_slots`
- 企画の基本情報（企画名、場所）を表示
- 時間枠ごとの定員は `events.capacity_per_slot` を使用
- 残り枠は $\text{定員}-\text{発券数}$ で算出
  - 発券数は `time_slots.current_count`
- 配布率（%）を表示
  - $\text{配布率(\%)}=\left\lfloor\frac{\text{発券数}}{\text{定員}}\times100\right\rfloor$
- 受付終了枠は「受付終了」表示

### 3. 共通機能

- 企画ごとのカード表示（配布方式・数値・更新時刻）
- 表示フィルタ（順次案内のみ / 定員制のみ / 全て）
- データ更新は自動（30秒ごと）で反映
- エラー時は「データを取得できませんでした」を表示
- 本システムではデータの編集・更新は行わない

## 画面設計

### 1. 配布率ダッシュボード

**画面名:** TicketDashboardScreen
**パス/識別子:** `/tickets/dashboard`

**表示項目:**

- 画面タイトル
- フィルタ切替（全て / 順次案内 / 定員制）
- 企画カード一覧

**カード（順次案内制）:**

- 企画名
- 受付済み人数
- 待ち人数
- 推定待ち時間
- 最終更新時刻

**カード（定員制）:**

- 企画名
- 定員
- 配布済み
- 残り枠
- 配布率
- 最終更新時刻

**動作:**

1. 画面読み込み時に Supabase から最新データ取得
2. 30秒ごとに自動再取得
3. 取得失敗時はエラーメッセージ表示

## データベース設計

### テーブル一覧（既存・追加作成なし）

| テーブル名 | 説明 |
| --- | --- |
| events | 企画情報（タイプ/待ち時間/定員） |
| event_dates | 企画開催日（発券ステータス/次の番号） |
| time_slots | 時間枠（時間枠定員制のみ） |
| tickets | 発券済み整理券（番号） |
| call_status | 呼び出し状態（順次案内制のみ） |
| ticket_logs | 発券ログ（表示には使用しない） |

### events テーブル（抜粋）

| カラム名 | 型 | 説明 |
| --- | --- | --- |
| id | uuid | 企画ID |
| name | text | 企画名 |
| location | text | 企画場所 |
| type | text | `time_slot` または `sequential` |
| capacity_per_slot | integer | 1枠あたり定員（time_slotのみ） |
| estimated_wait_minutes | integer | 1番号あたり待ち時間（sequentialのみ） |

### event_dates テーブル（抜粋）

| カラム名 | 型 | 説明 |
| --- | --- | --- |
| id | uuid | 開催日ID |
| event_id | uuid | 企画ID |
| date | date | 開催日 |
| status | text | `active` などのステータス |
| next_ticket_number | integer | 次の整理番号（最後尾+1） |

### time_slots テーブル（抜粋）

| カラム名 | 型 | 説明 |
| --- | --- | --- |
| id | uuid | 時間枠ID |
| event_id | uuid | 企画ID |
| event_date_id | uuid | 開催日ID |
| start_time | time | 開始時刻 |
| end_time | time | 終了時刻 |
| status | text | 枠ステータス |
| current_count | integer | 発券済み数 |

### call_status テーブル（抜粋）

| カラム名 | 型 | 説明 |
| --- | --- | --- |
| id | uuid | 呼び出し状態ID |
| event_id | uuid | 企画ID |
| event_date_id | uuid | 開催日ID |
| current_call_number | integer | 現在の呼び出し番号 |

## API 設計

### Supabase Database API

**企画と開催日を取得:**
```javascript
supabase
  .from('events')
  .select(`
    *,
    event_dates (
      id,
      date,
      status,
      next_ticket_number,
      time_slots (
        id,
        start_time,
        end_time,
        status,
        current_count
      )
    )
  `)
```

**呼び出し状態を取得（順次案内制）:**
```javascript
supabase
  .from('call_status')
  .select('*')
  .eq('event_id', eventId)
  .eq('event_date_id', eventDateId)
```
