# プロジェクト仕様書 — Item4: 落とし物検索

## 概要

渉外部が管理する落とし物スプレッドシートの情報を、ERP上で見やすく閲覧できる機能。
データの登録・編集は行わず、**閲覧（検索・フィルタ）のみ**の読み取り専用機能。

### 基本情報

| 項目 | 内容 |
|------|------|
| 機能名 | 落とし物検索 |
| フォルダ | `src/features/item4/` |
| Drawer表示名 | 落とし物検索 |
| 権限名（PERMISSION_NAME_MAP） | 落とし物検索 |
| アクセス権限 | 全ユーザー（ログイン済み） |
| データソース | Google スプレッドシート（CSV直接取得） |

### 環境変数

| 変数名 | 説明 |
|--------|------|
| `EXPO_PUBLIC_LOSTITEM_SPREADSHEET_ID` | 落とし物管理スプレッドシートのID |
| `EXPO_PUBLIC_LOSTITEM_DRIVE_FOLDER_ID` | 落とし物写真保管 Google Drive フォルダのID |

## データソース

### スプレッドシート構成

1つのスプレッドシートに3つのシートが存在する。

#### シート1: 「一般」（通常の落とし物）

| 列 | 内容 | 表示 | 備考 |
|----|------|------|------|
| A | 識別タグ | ○ | 例: `N-001`, `B-002` |
| B | 写真 | △（試験的） | `=IMAGE("URL")` 形式。URLからファイルIDを抽出して表示を試みる。Google Drive の制約により表示できない場合はプレースホルダーを表示 |
| C | 拾得物の名前 | ○ | |
| D | 拾得時間 | ○ | `yyyy/mm/dd hh:mm` 形式 |
| E | 拾得場所 | ○ | |
| F | 学籍番号 | **×（非表示）** | 個人情報のため |
| G | 返却日 | ○ | `yyyy/mm/dd hh:mm` 形式。空欄 = 保管中 |

#### シート2: 「緊急」（重要性の高い落とし物）

| 列 | 内容 | 表示 | 備考 |
|----|------|------|------|
| A | 識別タグ | ○ | 例: `N赤-001` |
| B | 写真 | △（試験的） | 同上 |
| C | 拾得物の名前 | ○ | |
| D | 拾得時間 | ○ | `yyyy/mm/dd hh:mm` 形式 |
| E | 拾得場所 | ○ | |
| F | 学籍番号 | **×（非表示）** | 個人情報のため |
| G | 返却日 | ○ | `yyyy/mm/dd hh:mm` 形式。空欄 = 保管中 |

#### シート3: 「落とし主」（紛失物の問い合わせ）

| 列 | 内容 | 表示 | 備考 |
|----|------|------|------|
| A | 識別番号 | ○ | 連番 |
| B | 紛失物の名前 | ○ | |
| C | 落とした可能性のある場所 | ○ | |
| D | 気づいた時間 | ○ | `yyyy/mm/dd hh:mm` 形式 |
| E | 連絡先 | **×（非表示）** | 個人情報のため |
| F | 学籍番号 | **×（非表示）** | 個人情報のため |
| G | 紛失者名 | **×（非表示）** | 個人情報のため |
| H | 返却日 | ○ | `yyyy/mm/dd hh:mm` 形式。空欄 = 未対応 |

### 写真データ

- Google Drive の特定フォルダに保管
- ファイル命名規則: `lostitem_{timestamp}.jpg`
- フォルダは「リンクを知っている人全員が閲覧可能」に設定済み
- 表示URL形式: `https://drive.google.com/thumbnail?id={fileId}&sz=w400`

**⚠️ 写真表示の制約事項**

Google Drive の仕様変更により、外部アプリからの画像埋め込みが制限される場合がある。

- 実装方針: `onError` でフォールバックを表示する形で試験的に実装する
- `<Image>` コンポーネントの `onError` コールバックでプレースホルダー画像に切り替え
- 実際にテストして表示できない場合は**写真列を非表示とし、「写真は現地でご確認ください」などのテキストに差し替える**

### データ取得方法

`gasApi.js` の `fetchShiftData` と同パターンで、Google Sheets CSV公開URLから直接取得。

```
URL形式: https://docs.google.com/spreadsheets/d/{SPREADSHEET_ID}/gviz/tq?tqx=out:csv&sheet={シート名}
```

- GASデプロイ不要
- CORS問題なし
- CSVレスポンスをパースして2次元配列に変換

## 機能要件

### 1. タブ切り替え

- 3つのタブ: 「一般」「緊急」「落とし主」
- タブ切り替え時にデータを再取得（またはキャッシュから表示）
- アクティブタブの視覚的ハイライト

### 2. テキスト検索

- 検索対象:
  - 一般・緊急: 識別タグ、拾得物名、拾得場所
  - 落とし主: 識別番号、紛失物名、場所
- リアルタイムフィルタリング（入力に応じて即時絞り込み）
- 検索キーワードのクリアボタン

### 3. ステータスフィルタ

- フィルタ状態:
  - 「すべて」（デフォルト）
  - 「保管中」/ 「未対応」（返却日が空のもの）
  - 「返却済み」/ 「対応済み」（返却日が入っているもの）
- テキスト検索と併用可能

### 4. 落とし物表示（一般・緊急共通）

- カード形式で一覧表示
- 表示項目: 識別タグ、写真サムネイル、拾得物名、拾得時間、拾得場所、ステータス（保管中/返却済み）
- 緊急タブの場合はカードに「緊急」バッジを表示
- 返却済みの場合は返却日も表示
- 写真: 表示を試み、失敗時はプレースホルダー（「写真なし」アイコン）を表示。写真タップで拡大表示（表示できている場合のみ）

### 5. 落とし主表示

- カード形式で一覧表示
- 表示項目: 識別番号、紛失物名、場所、気づいた時間、ステータス（未対応/対応済み）
- 対応済みの場合は返却日も表示

### 6. データ更新

- 画面表示時に自動取得
- プルダウン（下に引っ張る）で手動リフレッシュ
- ローディングインジケーター表示

### 7. 個人情報保護

- 一般・緊急シート: F列（学籍番号）を一切表示しない
- 落とし主シート: E列（連絡先）、F列（学籍番号）、G列（紛失者名）を一切表示しない
- CSVパース時に該当列を除外してメモリに保持しない

## 画面設計

### Item4Screen（落とし物検索メイン画面）

**画面名:** Item4Screen
**Drawer表示名:** 落とし物検索

**レイアウト（上から順）:**

1. **ThemedHeader** — 「落とし物検索」
2. **SearchBar** — テキスト検索入力欄（プレースホルダー: 「名前や場所で検索」）
3. **StatusFilter** — 「すべて」「保管中」「返却済み」のセグメントボタン
4. **TabSelector** — 「一般」「緊急」「落とし主」タブ
5. **コンテンツエリア** — タブに応じたリスト表示
   - 一般タブ → LostItemList（LostItemCardの一覧）
   - 緊急タブ → LostItemList（LostItemCardの一覧 + 緊急バッジ）
   - 落とし主タブ → OwnerInquiryList（OwnerInquiryCardの一覧）

**動作:**

1. **初期表示**
   - 画面を開くとスプレッドシートから3シート分のCSVデータを取得
   - パース後、個人情報列を除外した状態でstateに保持
   - デフォルトは「一般」タブ、「すべて」フィルタ
   - ローディングインジケーター表示中

2. **タブ切り替え**
   - タブをタップすると表示データが切り替わる
   - 検索キーワードとフィルタ状態はタブ間で保持される

3. **検索操作**
   - テキスト入力に応じてリアルタイムに絞り込み
   - 検索は名前・場所・タグ等を横断的にマッチ
   - ×ボタンで検索クリア

4. **フィルタ操作**
   - 「保管中」選択 → 返却日が空のアイテムのみ表示
   - 「返却済み」選択 → 返却日が入っているアイテムのみ表示
   - 「すべて」選択 → フィルタ解除

5. **写真タップ**
   - サムネイル画像をタップ → モーダルで拡大表示

6. **プルリフレッシュ**
   - 下に引っ張る → スプレッドシートからデータ再取得

**エラーハンドリング:**

- CSV取得失敗時 → 「データの取得に失敗しました。再度お試しください。」エラー表示 + リトライボタン
- スプレッドシートID未設定時 → 「設定エラー: スプレッドシートIDが設定されていません」表示
- 写真読み込み失敗時 → プレースホルダー（「写真なし」アイコン）を表示。Google Drive の制約で表示不可と判明した場合は写真列ごと削除して「写真は現地でご確認ください」に変更
- データが0件の場合 → 「該当するデータがありません」空状態表示

    urgentItems,      // 緊急落とし物リスト

    setActiveTab,     // タブ切り替え
    searchQuery,      // 検索キーワード
    setSearchQuery,   // 検索キーワード更新
    statusFilter,     // ステータスフィルタ

    filteredItems,    // フィルタ適用済みの表示データ
    loading,          // ローディング状態

    refresh,          // データ再取得
