## プロジェクト概要

### 目的

生駒祭 2026 で使用する ERP（Enterprise Resource Planning）システムの設定画面に、
ライトモード・ダークモード・システム設定の切り替え機能を追加し、
PC とスマートフォンのどちらでも視認性の高い UI を提供する。

### スケジュール

- 開発開始：2026/01/30
- リリース予定：未定（他機能と合わせて調整）

## 対象ユーザー

### 運営（管理者）

- **いつ使うか：** 大学祭準備期間〜当日
- **どういうときに使うか：** ERP の設定画面で自分に合ったテーマを選択したいとき

- **いつ使うか：** 当日の業務中
- **どういうときに使うか：** スマホで画面を見やすくしたいとき、夜間作業で眩しさを抑えたいとき

## 機能要件

### 1. テーマ切り替え機能

**概要：**
ERP 設定画面から、以下のテーマに切り替えできる。

**詳細仕様：**

- 選択肢は「ライト」「ダーク」「女子大学生」「ワールドトリガー」「エヴァンゲリオン」の 5 つ
- 変更直後に画面全体のテーマを即時反映する
- 選択状態は次回起動時も保持する
- 既定値は「ライト」
- テーマ切り替え時は画面遷移なしで適用する

---

### 2. 設定の保存・復元

**概要：**
ユーザーごとにテーマ設定を保存し、ログイン時に復元する。

**詳細仕様：**

- ログインユーザーごとにテーマ設定を保存する
- 通信エラー時はローカルに保持した設定を優先して適用する
- ログアウト時はローカル設定を保持（次回ログイン時に上書きされる）

## 画面設計

### 1. 設定画面（テーマ設定）

**画面名:** SettingsThemeScreen  
**パス/識別子:** `/settings/theme`

**表示項目:**

- 画面タイトル（テキスト）
- テーマ選択（ラジオボタン）
  - ライトモード
  - ダークモード
  - 女子大学生モード
  - ワールドトリガーモード
  - エヴァンゲリオンモード
- 現在の適用状態（サブテキスト）

**動作:**

1. ユーザーがテーマを選択
2. 即時にテーマを適用
3. 保存処理を実行
4. 成功 → サブテキストに「保存しました」を表示
5. 失敗 → サブテキストに「保存に失敗しました（ローカルに保持）」を表示

## データベース設計

### テーブル一覧

| テーブル名 | 説明 |
| ---------- | ---- |
| users | Supabase Auth ユーザー（既存） |
| user_settings | ユーザーごとの設定を保存 |

---

### user_settings テーブル

**説明:** ユーザーごとのテーマ設定を保存

| カラム名 | 型 | 制約 | 説明 |
| --- | --- | --- | --- |
| user_id | uuid | PK, FK(users.id) | ユーザー ID |
| theme_mode | text | NOT NULL | `light` / `dark` / `joshi` / `world_trigger` / `eva` |
| updated_at | timestamptz | NOT NULL | 更新日時 |

**リレーション:**

- users.id と 1:1

## API 設計

### 1. Supabase REST API

**アクセス方法:**
ERP が Supabase に直接アクセスする場合

**ベース URL:**

```
https://{PROJECT_ID}.supabase.co/rest/v1
```

**認証方法:**

```
apikey: {SUPABASE_ANON_KEY}
Authorization: Bearer {SUPABASE_ANON_KEY}
Content-Type: application/json
```

---

#### テーマ設定取得

```
GET /user_settings?user_id=eq.{USER_ID}&select=*
```

**レスポンス例:**

```json
[
  {
    "user_id": "uuid",
    "theme_mode": "light",
    "updated_at": "2026-01-30T00:00:00Z"
  }
]
```

---

#### テーマ設定保存

```
UPSERT /user_settings
```

**リクエスト例:**

```json
{
  "user_id": "uuid",
  "theme_mode": "world_trigger",
  "updated_at": "2026-01-30T00:00:00Z"
}
```

## 非機能要件

### パフォーマンス

- テーマ切り替えは 300ms 以内に反映
- 設定保存 API の応答は 1 秒以内

### 可用性

- 祭期間中は稼働率 99%以上

### 対応環境

- **PC:** 768px 以上
- **スマートフォン:** 768px 未満

### アクセシビリティ

- ライト/ダークともに WCAG AA 相当のコントラストを確保
- テーマ選択はキーボード操作に対応

## セキュリティ設計

### 認証・認可

- **認証方式:** Supabase Auth（JWT）
- **権限:** ログインユーザーのみ設定変更可能

- **通信:** HTTPS のみ
- **API キー管理:** 環境変数（`.env`）
- **Supabase RLS:** user_settings は自分の行のみ閲覧・更新可能
