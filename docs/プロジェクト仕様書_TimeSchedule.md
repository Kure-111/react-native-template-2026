# プロジェクト仕様書 TimeSchedule

最終更新日: 2026/03/31
ステータス: 実装反映済み（現行仕様）

---

## 1. 概要

### 1.1 目的
大学祭当日の企画スケジュールを時系列で可視化し、日付ごとに開催状況を確認できる画面を提供する。

### 1.2 対象範囲
- 対象は `event_schedule_slots` に登録された `source_type=event` のみ（屋台は表示対象外）
- 運用時間（09:00〜20:00）内でタイムライン表示
- 表示対象はブックマーク（建物/エリア/団体）で切り替える

### 1.3 対象ユーザー
- 大学祭実行委員メンバー全員

## 2. 機能要件

### 2.1 日付切替
- 日付単位で時間表を切り替える
- 初期表示日は以下の優先順で決定する
  - `AsyncStorage` に保存された前回表示日（`time_schedule_selected_date`）
  - 環境変数 `EXPO_PUBLIC_FESTIVAL_START_DATE`（`YYYY-MM-DD` 形式のときのみ）
  - 端末の当日日付
- 表示可能日（`event_schedule_slots` に存在する開催日）に要求日付が存在しない場合、直近の過去日付へフォールバックする。過去日付がなければ最小日付へフォールバックする
- 画面上の日付移動は、表示可能日の配列に対して前日/翌日ボタンで行う

### 2.2 タイムライン表示
- 運用時間は 09:00〜20:00
- タイムラインの基準目盛りは15分刻み（09:00, 09:15, ... , 20:00）
- 1時間境界の罫線は強調表示し、それ以外は通常罫線で表示する
- 同時間帯で重なる企画は同一ブックマーク列内でレーン分割（横方向に増列）して重なりを回避する
- 該当企画がない列には「該当なし」バッジを表示する

### 2.3 表示対象の切替（ブックマーク方式）
- 画面上部に `デフォルト` / `マイリスト` タブを持つ
- `デフォルト` は建物ごとの固定ブックマーク（削除不可・表示/非表示のみ）
- `デフォルト`の項目は、
  - 11月ホール
  - 実学ホール
  - 記念会館
  - 人工芝グラウンド
  - その他

- `マイリスト` はユーザー作成ブックマーク（追加/編集/削除/並び替え/表示切替が可能）
- ブックマークの評価軸は以下
  - 建物
  - エリア
  - 団体
- 評価軸の候補検索は Fuse.js によるあいまい検索を使用する

### 2.4 ブックマーク編集
- 設定モーダルで以下を設定して保存する
  - 表示項目名
  - 評価軸
  - 評価軸候補（複数選択可）
  - テーマカラー（淡色パレット）
- デフォルトブックマークは名称/条件固定で、カラーのみ変更可能
- ブックマーク一覧モーダルは下書き編集方式
  - `×` または背景タップで閉じると下書きを破棄
  - `設定を反映する` で確定反映
- マイリストの並び替えはドラッグ&ドロップで行う

### 2.5 企画カード表示
- カード表示項目
  - 企画名
  - 開催時刻（`HH:mm - HH:mm`）
  - 開催場所（建物 + 場所）
- カードタップで詳細モーダルを開く
- 詳細モーダルは `01_Events&Stalls_list` の `DetailModal` を流用し、既存の企画詳細表示仕様と統一する

### 2.6 カードリストモード表示
- **時間表モード** に加え、新たに **カードリストモード** を追加
- カードリストモードの特徴
  - ブックマークを **単一選択** に変更（時間表モードでは複数選択可）
  - 時間軸を廃止し、時刻順にイベントを縦配列で表示
  - 横スクロール要素を完全廃止
  - イベントをカード形式で表示：企画名・時刻・場所・簡潔な詳細情報
  - カードタップで詳細モーダルを開く（時間表モードと同様）
- 参考UI: 本部機能（item10）の「本日のタイムスケジュール」コンポーネント

### 2.7 表示モード切り替え
- 画面上部に **モード切り替えUI** を配置
- 選択肢: 「時間表」「リスト」 の 2 択
- モード選択は `AsyncStorage` に永続化（キー: `time_schedule_view_mode`）
- 初期表示モードは以下の優先順で決定
  - `AsyncStorage` の保存値（キー: `time_schedule_view_mode`）
  - 端末のデフォルト: 「時間表」

### 2.8 ブックマーク選択変更
- **時間表モード**
  - デフォルト/マイリスト タブで分離
  - 複数ブックマークの表示/非表示を切り替え可能
  - ブックマーク編集UI（追加/削除/並び替え）利用可
- **カードリストモード**
  - ドロップダウン形式で **単一ブックマーク選択**
  - デフォルト/マイリスト の区別を簡略化（統合ドロップダウン）
  - ブックマーク編集UI は画面上部の設定ボタンから別途アクセス

### 2.9 更新方式
- データ取得は以下タイミングで実行する
  - 初回表示
  - 日付変更時
  - 表示モード切り替え時
- 定期自動更新は行わない
- 取得失敗時のみエラー表示と `再取得` ボタンを表示する

## 3. 画面要件

### 3.1 画面名
- タイムスケジュール（`SCREEN_NAME`）

### 3.2 基本構成（時間表モード）
- ヘッダー
- 日付切替行（前日/翌日）
- **モード切り替えタブ**（「時間表」「リスト」）
- 表示タブ（デフォルト/マイリスト）
- 設定導線（PCは上部ボタン、モバイルはフローティングボタン）
- タイムライン本体（縦スクロール + 横スクロール）

### 3.3 基本構成（カードリストモード）
- ヘッダー
- 日付切替行（前日/翌日）
- **モード切り替えタブ**（「時間表」「リスト」）
- **ブックマーク単一選択ドロップダウン**
- 設定導線（PCは上部ボタン、モバイルはフローティングボタン）
- イベントカードリスト（縦スクロールのみ）

### 3.4 レスポンシブ
- ブレークポイント: 768px 未満をモバイルレイアウトとして扱う
- モバイルはフローティング設定ボタンを表示
- PCはタブ行の右に設定ボタンを表示

## 4. データ要件（両モード共通）

### 4.1 参照テーブル
- `event_schedule_slots`
- `events`
- `building_locations`
- `area_locations`
- `event_organizations`

### 4.2 `event_schedule_slots` の利用条件
- `source_type='event'` のみ取得対象
- 基本条件
  - `schedule_date = 選択日`
  - `is_visible_time_schedule = true`（列が存在しない環境では条件なしフォールバック）
- スキーマ差分互換のため、`area_code` がない環境では `area_name` を利用するフォールバックを持つ

### 4.3 ソース詳細結合
- `source_id` を `events.id` に結合して表示データを生成する
- `events` 取得時は select 候補を段階的にフォールバックし、環境差分で一部列やリレーションが欠ける場合でも可能な範囲で表示継続する
- `schedule_dates` / `schedule_start_times` / `schedule_end_times` が空の場合は、スロット日付・時刻から補完する

### 4.4 表示順と投影
- **時間表モード**: タイムライン投影判定 `start_time <= slot_time < end_time`
- **カードリストモード**: 選択ブックマーク条件に合致するイベントを時刻昇順で表示
- 各イベント内での表示順
  - 開始時刻昇順
  - 同時刻は表示名昇順（日本語ロケール、数値比較あり）

## 6. ローカル保存要件

### 6.1 保存項目
- 選択日付
- マイリストブックマーク一覧
- アクティブブックマークID（時間表モード）
- 非表示ブックマークID一覧（時間表モード）
- ブックマーク並び順ID一覧（時間表モード）
- デフォルトブックマークの色設定
- **表示モード**（新規：時間表 or リスト）
- **カードリストモード用アクティブブックマークID**（新規：カードリストモード用の単一選択）

### 6.2 保存スコープ
- 選択日付は共通キー（全ユーザー共通キー）
- 表示モード は共通キー（全ユーザー共通キー）
- ブックマーク関連はユーザーID付きキーで保存（ユーザーごとに分離）
