## テストシナリオ例

### 正常系

- 出展団体が緊急連絡を送信 → 本部に表示される
- 巡回が「向かいます」→「完了」→ 結果が本部に反映される
- 鍵返却処理 → 施錠確認タスクが生成される

### 異常系

- ネットワーク切断時のエラーハンドリング
- 不正なデータ入力時のバリデーション
- 権限のないロールがアクセスした場合の制限

### 境界値

- 長文テキストの入力
- 同時アクセス（複数デバイスからの操作）
- 大量データ時のパフォーマンス

## 注意事項

- テストは仕様書に基づいて設計する
- バグを発見したら、再現手順を明確に記録する
- テストデータは本番データを使わない
- RLS のテストを必ず含める（権限外アクセスの確認）
- 破壊的テスト（データ削除等）はテスト環境でのみ実施する
