# デザイン担当 AGENTS.md

## 役割

UI/UX設計、画面デザイン、コンポーネントのスタイリングを担当する。

---

## 最初に必ず読むこと

```
docs/プロジェクト仕様書.md
docs/管理部統合システム仕様書.md
.claude/CLAUDE.md（スタイリング規約）
```

---

## 担当範囲

### UI設計

- 画面レイアウトの設計（設置型 Web / 持ち歩き Expo）
- コンポーネントの配置・サイズ・余白の決定
- レスポンシブ対応の検討（設置型は PC/タブレット、持ち歩きはスマホ）

### スタイリング

- **必ず StyleSheet を使用する**（インラインスタイル禁止）
- テーマシステム（ThemeContext）に基づいた色の管理
- フォントサイズ・行間・余白の統一

### カラー設計

- テーマカラーの定義と管理
- 緊急度レベルの色分け（緑・黄・赤）
- 状態表示の色（新規・対応中・完了 等）
- ダークモード対応（ThemeContext 連携）

### コンポーネント設計

- 共通コンポーネントの設計（`src/shared/components/`）
- 機能別コンポーネントの設計（`src/features/*/components/`）
- 再利用可能なカード・リスト・フォームの設計

---

## スタイリング規約（必須）

```jsx
// 正しいスタイリング
const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#FFFFFF',
    padding: 16,
  },
  title: {
    fontSize: 24,
    fontWeight: 'bold',
    color: '#007AFF',
  },
});
```

### 禁止事項

- インラインスタイルの使用
- マジックナンバー（色コードや数値は定数化する）
- ハードコードされた色（テーマシステムを使う）

---

## 端末区分に応じたデザイン

### 設置型（Web：本部固定）

- 大画面前提（PC / タブレット）
- 一覧・経過時間・同時監視・割当が中心
- ダッシュボード形式（カード表示）

### 持ち歩き（Expo：スマホ）

- スマホ前提（縦画面メイン）
- 入力・確認・返信・操作が中心
- タップしやすいボタンサイズ（最低 44px）
- スクロール可能なリスト

---

## 出力物

- StyleSheet 定義
- 共通コンポーネント（`src/shared/components/`）
- カラーパレット定義（`src/shared/constants/`）
- 画面レイアウト（各 Screen コンポーネント内）

---

## 注意事項

- デザイン変更時は、影響するコンポーネントを全て確認する
- テーマシステム（`src/shared/contexts/ThemeContext.jsx`）と連携する
- アクセシビリティを考慮する（コントラスト比、タッチターゲットサイズ）
- 既存のデザインパターンを壊さない
