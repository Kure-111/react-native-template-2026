# フォント・アイコン追加ルール

本プロジェクトでは、Cloudflare Pagesへのデプロイ環境における既知のバグ（`node_modules` 配信制限）を回避するため、フォントやアイコンの追加手順に特別なルールを設けています。
他のAIエージェントや開発者はフォント・アイコンを追加する際、**必ず以下の手順を遵守してください。**

## 🚨 絶対厳守：やってはいけないこと

- **`@expo/vector-icons` からの直接インポートは禁止です。**
  - ❌ `import { FontAwesome } from '@expo/vector-icons';`
  - ❌ `import FontAwesome from '@expo/vector-icons/FontAwesome';`
- 上記を行うと、Expoのビルド（Metro Bundler）がフォントファイルを `dist/assets/node_modules/...` に出力してしまいます。
- Cloudflare Pagesはセキュリティ/容量削減の独自仕様により、**名前に `node_modules` が含まれるディレクトリをインターネット上（CDN）に一切アップロードしません。**
- 結果として、ローカル環境では動くのに、本番環境ではフォントファイルが見つからず（404エラー）、表示が四角（豆腐）になったり、「Failed to decode downloaded font」「OTS parsing error」などの致命的なエラーが発生します。

## ✅ 正しいフォント・アイコンの追加手順（Cloudflare回避版）

新しいアイコンセット（例: `FontAwesome`）を追加する場合は、以下の手順で**フォントファイルをプロジェクト内にローカル配置**し、自前のラッパーを通して読み込みます。

### 1. フォントファイル（.ttf）のコピー配置
使用したいアイコンの `.ttf` ファイルを `node_modules` 内から探し、プロジェクト内の `assets/fonts/` フォルダへ手動でコピー（配置）します。

```bash
# 例: FontAwesome を追加する場合
cp node_modules/@expo/vector-icons/build/vendor/react-native-vector-icons/Fonts/FontAwesome.ttf assets/fonts/
```

### 2. カスタムアイコンラッパーへの定義追加
`src/shared/components/icons/index.js` を開き、新しく配置したローカルのフォントファイルを読み込む定義を追加します。

```javascript
// src/shared/components/icons/index.js の編集例

import createIconSet from '@expo/vector-icons/createIconSet';
// 1. GlyphMap(JSON) は @expo/vector-icons 側からimportしてOK（テキストデータなので問題なし）
import fontAwesomeGlyphMap from '@expo/vector-icons/build/vendor/react-native-vector-icons/glyphmaps/FontAwesome.json';

// 2. ★重要★ フォントファイルの実体(.ttf)は、必ずローカルの assets/fonts/ から require() する
const fontAwesomeFont = require('../../../../assets/fonts/FontAwesome.ttf');

// ... 既存の定義 ...

// 3. createIconSet を使ってエクスポートする
export const FontAwesome = createIconSet(fontAwesomeGlyphMap, 'FontAwesome', fontAwesomeFont);
```

### 3. FontLoaderProviderへの追加
アプリ起動時にフォントを確実に読み込ませるため、`src/shared/components/FontLoaderProvider.jsx` の `useFonts` に新しいフォントを追加します。

```javascript
// src/shared/components/FontLoaderProvider.jsx の編集例
export const FontLoaderProvider = ({ children }) => {
    const [fontsLoaded, fontError] = useFonts({
        Ionicons: require('../../../assets/fonts/Ionicons.ttf'),
        MaterialCommunityIcons: require('../../../assets/fonts/MaterialCommunityIcons.ttf'),
        // 新しくローカルからrequireして追加する
        FontAwesome: require('../../../assets/fonts/FontAwesome.ttf') 
    });
    // ...
```

### 4. アプリ内での使用方法
画面やコンポーネントでアイコンを使用する場合は、`@expo/vector-icons` ではなく、**必ず作成した内製ラッパーから import してください。**

```javascript
// ❌ ダメな例（ビルド時に node_modules パスが生成されてしまう）
// import { FontAwesome } from '@expo/vector-icons';

// ✅ 正しい例（カスタムラッパーからインポートする）
import { FontAwesome } from '../../../shared/components/icons';

const MyComponent = () => {
    return <FontAwesome name="check" size={24} color="black" />;
}
```

---
このルールを守ることで、Expo(Metro)のビルド出力パスに `node_modules` が含まれなくなり、Cloudflare Pages へ正常に実体ファイルが配信（デプロイ）されるようになります。
