## 2. UI/UX 評価と改善指示

### 2.1 全画面共通の問題（優先度: 高）

#### 問題1: ローディング表示が貧弱

- **現状:** 「読み込み中...」のテキストのみ
- **改善案:** スケルトンローダーまたはスピナーを導入する
- **担当:** デザイン担当 → フロントエンド担当
- **理由:** ユーザーに「動いている」という安心感を与える

#### 問題2: 空状態の表示が不親切

- **現状:** テキストのみ（例:「タスクはありません」）
- **改善案:** アイコン + 説明文 + アクション導線を追加
- **担当:** デザイン担当 → フロントエンド担当
- **例:**
  ```
  [アイコン]
  タスクはありません
  新しいタスクが割り当てられると、ここに表示されます。
  [更新する]
  ```

#### 問題3: 確認ダイアログが不足

- **現状:** タスク完了・鍵返却など重要操作に確認なし
- **改善案:** 「本当に完了しますか？」等の確認ダイアログを追加
- **担当:** フロントエンド担当
- **対象操作:**
  - タスクの「向かいます」「完了」
  - 鍵の返却処理
  - 連絡案件のステータス変更
  - 評価の承認/差戻し

#### 問題4: ドロワーメニューにアイコンがない

- **現状:** テキストのみのメニュー
- **改善案:** 各メニュー項目にアイコンを追加して視認性を向上
- **担当:** デザイン担当 → フロントエンド担当

## 3. 不足機能の提案

### 優先度: 高（当日運用に必要）

#### 提案1: プッシュ通知 / アプリ内通知

- **現状:** 通知機能が未実装（`notificationService.js` が「将来的に」のまま）
- **必要性:** 緊急連絡・新着タスク・ステータス変更をリアルタイムで各ロールに知らせる必要がある
- **実装案:**
  - Expo Push Notifications を使ったプッシュ通知
  - アプリ内通知バッジ（ドロワーメニューに未読数を表示）
  - 通知設定画面（ON/OFF切替）
- **担当:** バックエンド担当（Edge Function + 通知ログ） → フロントエンド担当（表示）

#### 提案2: オフライン対応（下書き保存）

- **現状:** ネットワーク切断時にデータが失われる
- **必要性:** 学祭会場はWi-Fi不安定。巡回中の電波が弱いエリアでもデータを失わない
- **実装案:**
  - AsyncStorage による下書き自動保存
  - オンライン復帰時に自動送信
  - オフライン状態の警告バナー
- **担当:** フロントエンド担当
- **対象:** 巡回チェック、緊急連絡、企画報告

#### 提案3: 経過時間アラート（遅延警告）

- **現状:** 経過時間は表示されるが、閾値超過のアラートがない
- **必要性:** 対応漏れを防ぐために、長時間未対応の案件を目立たせる必要がある
- **実装案:**
  - 設定可能な閾値（例: 15分で黄色、30分で赤）
  - ダッシュボードに「遅延案件」セクションを追加（本部側）
  - 音声/バイブレーション通知（オプション）
- **担当:** フロントエンド担当 → デザイン担当（色設計）

### 優先度: 中（品質向上）

#### 提案4: 写真プレビュー・拡大機能

- **現状:** 添付画像は小さいプレビューのみ
- **改善案:** タップでフルスクリーン表示、ピンチで拡大縮小
- **担当:** フロントエンド担当

#### 提案5: 検索機能

- **現状:** 連絡案件のフリーテキスト検索がない
- **改善案:** 受付番号・件名・場所でのインクリメンタル検索
- **担当:** フロントエンド担当 → バックエンド担当（全文検索インデックス）

#### 提案6: 統計・集計ダッシュボード

- **現状:** 本部に集計表示がない
- **改善案:**
  - 種別ごとの案件数（円グラフ）
  - 時間帯別の受付数（棒グラフ）
  - 平均対応時間
  - 巡回カバー率
- **担当:** フロントエンド担当 → デザイン担当

#### 提案7: CSV/レポートエクスポート

- **現状:** データはSupabaseに蓄積されるが、エクスポート手段がない
- **改善案:** 当日終了後の振り返り用にCSVダウンロード機能を追加
- **担当:** バックエンド担当（GAS連携でスプレッドシート出力も検討）
