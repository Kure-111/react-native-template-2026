# プロジェクト仕様書 - 初回ログインパスワード変更システム

## 概要

ユーザーが管理者から付与された仮パスワードで初回ログインした際に、パスワード変更を促すシステム。セキュリティを確保しつつ、ユーザビリティにも配慮した任意変更方式を採用。

---

## 機能要件

### 基本方針

- **任意変更方式**: 初回ログイン時にパスワード変更を推奨するが、スキップ可能
- **後から変更可能**: スキップした場合でも、プロフィール画面などから変更可能
- **セキュリティとユーザビリティのバランス**: 強制しないが、変更を促す

### パスワードポリシー

- **最小文字数**: 8文字以上
- **複雑性要件**: なし（英数字記号の組み合わせは不問）
- **仮パスワード禁止**: 仮パスワードと同じパスワードは設定不可（推奨）

---

## 認証フロー

### 1. ログイン時の判定フロー

```
1. ユーザーがメールアドレスとパスワードでログイン
   ↓
2. Supabase Auth API で認証
   ↓
3. 認証成功後、user_profiles.password_changed_at を確認
   ↓
4a. password_changed_at が NULL（初回ログイン）
    → パスワード変更推奨モーダル/画面を表示
    → ユーザーは「今すぐ変更」または「後で変更」を選択可能
   ↓
4b. password_changed_at に値がある（2回目以降のログイン）
    → 通常通りメイン画面に遷移
```

### 2. パスワード変更フロー

```
1. ユーザーが「今すぐ変更」を選択、またはプロフィール画面から変更を開始
   ↓
2. パスワード変更画面を表示
   - 現在のパスワード入力
   - 新しいパスワード入力
   - 新しいパスワード（確認用）入力
   ↓
3. バリデーション
   - 現在のパスワードが正しいか確認
   - 新しいパスワードが8文字以上か確認
   - 新しいパスワードと確認用が一致するか確認
   - （任意）仮パスワードと異なるか確認
   ↓
4. Supabase Auth API でパスワード更新
   ↓
5. user_profiles.password_changed_at を現在時刻で更新
   ↓
6. 成功メッセージを表示
   ↓
7. 当日部員シフト確認画面（jimu-shift）に遷移
```

### 3. スキップ時の動作

```
1. ユーザーが「後で変更」を選択
   ↓
2. password_changed_at は NULL のまま（変更しない）
   ↓
3. 通常通りメイン画面に遷移
   ↓
4. 次回ログイン時も再度パスワード変更推奨モーダルが表示される
```

## データベース設計

### user_profiles テーブルへのカラム追加

| カラム名 | 型 | 説明 | 制約 |
|---------|-----|------|------|
| password_changed_at | TIMESTAMPTZ | パスワード最終変更日時 | DEFAULT NULL |

**追加SQL:**
```sql
ALTER TABLE user_profiles
ADD COLUMN password_changed_at TIMESTAMPTZ DEFAULT NULL;
```

**初回ログインの判定ロジック:**
```javascript
// password_changed_at が NULL の場合、初回ログインと判定
const isFirstLogin = user.password_changed_at === null;
```

**パスワード変更後の更新:**
```javascript
// パスワード変更成功後、現在時刻を記録
await supabase
  .from('user_profiles')
  .update({ password_changed_at: new Date().toISOString() })
  .eq('user_id', userId);
```

## バリデーション

### クライアント側バリデーション

1. **現在のパスワード**
   - 空欄チェック
   - エラーメッセージ: "現在のパスワードを入力してください"

2. **新しいパスワード**
   - 空欄チェック → "新しいパスワードを入力してください"
   - 文字数チェック（8文字未満） → "パスワードは8文字以上で設定してください"
   - （任意）現在のパスワードと同じ → "現在のパスワードと異なるパスワードを設定してください"

3. **新しいパスワード（確認用）**
   - 空欄チェック → "確認用パスワードを入力してください"
   - 新しいパスワードと不一致 → "パスワードが一致しません"

### サーバー側バリデーション（Supabase）

1. **現在のパスワードの検証**
   - Supabase Auth API で現在のパスワードが正しいか確認
   - エラーメッセージ: "現在のパスワードが正しくありません"

2. **パスワード更新**
   - Supabase Auth API の `updateUser` を使用
   - エラーハンドリング: API エラーをキャッチして適切なメッセージを表示

## API 設計

### Supabase Auth API

**パスワード更新:**
```javascript
// 現在のパスワードで再認証（オプション）
await supabase.auth.signInWithPassword({
  email: user.email,
  password: currentPassword
});

// パスワード更新
const { data, error } = await supabase.auth.updateUser({
  password: newPassword
});

if (error) {
  throw new Error('パスワードの更新に失敗しました');
}
```

### Supabase Database API

**password_changed_at の更新:**
```javascript
const { data, error } = await supabase
  .from('user_profiles')
  .update({ password_changed_at: new Date().toISOString() })
  .eq('user_id', userId);

if (error) {
  throw new Error('パスワード変更日時の記録に失敗しました');
}
```

**初回ログイン判定:**
```javascript
const { data: profile, error } = await supabase
  .from('user_profiles')
  .select('password_changed_at')
  .eq('user_id', userId)
  .single();

const isFirstLogin = profile?.password_changed_at === null;
```

## エラーハンドリング

### エラーケースとメッセージ

| エラーケース | エラーメッセージ | 対処方法 |
|-------------|----------------|---------|
| 現在のパスワードが空欄 | "現在のパスワードを入力してください" | 入力フィールドに赤枠表示 |
| 新しいパスワードが空欄 | "新しいパスワードを入力してください" | 入力フィールドに赤枠表示 |
| 新しいパスワードが8文字未満 | "パスワードは8文字以上で設定してください" | 入力フィールドに赤枠表示 |
| 確認用パスワードが不一致 | "パスワードが一致しません" | 確認用フィールドに赤枠表示 |
| 現在のパスワードが誤り | "現在のパスワードが正しくありません" | 現在のパスワードフィールドに赤枠表示 |
| Supabase API エラー | "パスワードの更新に失敗しました。もう一度お試しください。" | エラーメッセージを画面上部に表示 |
| ネットワークエラー | "ネットワークエラーが発生しました。接続を確認してください。" | エラーメッセージを画面上部に表示 |

### エラー表示方法

- **入力エラー**: 該当フィールドの下に赤字で表示 + フィールドに赤枠
- **システムエラー**: 画面上部に赤い背景のアラートで表示
- **エラーアイコン**: ⚠️ を使用して視認性を高める

## 実装の流れ

### フェーズ1: データベース準備

1. user_profiles テーブルに password_changed_at カラムを追加
2. 既存ユーザーの password_changed_at は NULL のままにする（初回ログイン扱い）

### フェーズ2: UI コンポーネント作成

1. PasswordChangeModal コンポーネント（推奨モーダル）
2. PasswordChangeForm コンポーネント（変更フォーム）
3. PasswordSuccessModal コンポーネント（成功通知）
4. ChangePasswordScreen コンポーネント（変更画面）

### フェーズ3: ロジック実装

1. usePasswordChange カスタムフック
   - パスワード変更処理
   - バリデーション
   - エラーハンドリング
2. passwordService.js
   - Supabase Auth API 呼び出し
   - password_changed_at 更新

### フェーズ4: 認証フロー統合

1. AuthContext に初回ログイン判定を追加
2. ログイン成功後、password_changed_at をチェック
3. NULL の場合、PasswordChangeModal を表示

### フェーズ5: テスト

1. 初回ログイン時の動作確認
2. パスワード変更成功時の動作確認
3. スキップ時の動作確認
4. バリデーションの動作確認
5. エラーケースの動作確認

## セキュリティ考慮事項

1. **パスワードの送信**
   - HTTPS 必須（本番環境）
   - Supabase が TLS で暗号化

2. **パスワードの保存**
   - フロントエンドでパスワードを保存しない
   - Supabase Auth が bcrypt でハッシュ化して保存

3. **現在のパスワード検証**
   - 再認証を行い、本人確認を徹底

4. **セッション管理**
   - パスワード変更後、セッションは維持（ログアウトしない）
   - Supabase が自動的に新しいトークンを発行

5. **エラーメッセージ**
   - セキュリティ上の理由で、詳細すぎる情報は表示しない
   - 例: "メールアドレスまたはパスワードが正しくありません"（どちらが誤りかは明示しない）

## 将来的な拡張

1. **パスワード強度メーター**
   - リアルタイムでパスワードの強度を表示
   - 弱い/普通/強いの3段階評価

2. **パスワード履歴**
   - 過去3回分のパスワードは再利用不可
   - user_profiles に password_history カラム（JSONB）を追加

3. **2要素認証（2FA）**
   - メール認証、SMS認証、Google Authenticatorなど
   - Supabase が標準でサポート

4. **パスワードリセット機能**
   - メールでリセットリンクを送信
   - Supabase Auth の resetPasswordForEmail を使用

5. **パスワード変更の通知**
   - パスワード変更時にメールで通知
   - 不正アクセスの検知に役立つ
