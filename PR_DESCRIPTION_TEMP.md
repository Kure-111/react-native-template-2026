## 概要
Web（Cloudflare Pages）本番環境において、アイコン（`@expo/vector-icons`）が文字化け・エラー表示になる問題（OTS parsing error / 404エラー）を根本から解決するためのリファクタリングを行いました。

## 課題の背景と原因の特定
調査の結果、本番環境でのアイコン表示エラーには主に**3つの複合的な原因**があることが判明しました。

1. **ツリーシェイキングの不備による大量ダウンロード**
   `@expo/vector-icons` からバレルインポート（中括弧でのインポート）を行っていたため、使用していないフォント（20種類以上）まで全てブラウザがダウンロードしようとし、ネットワークを逼迫・ブロックしていました。
2. **SPAにおける相対パス解決のバグ**
   React Navigation等でサブルート（例：`/item10`）へ遷移した状態でリロードすると、ブラウザがフォントの相対パスを誤認識し（例：`/item10/assets/...`）、404エラーを引き起こしていました。
3. **Cloudflare Pages 特有の `node_modules` 配信ブロック（最大の原因）**
   Expo (Metro) はWebビルド時、フォントの実体を `dist/assets/node_modules/...` というディレクトリに出力します。しかし、**Cloudflare Pagesのアップロード仕様上「名前に `node_modules` が含まれるディレクトリはビルド成果物から強制的に配信から除外する」**というセキュリティ仕様があり、そもそもCDN上にフォントファイルが存在しない状態になっていました。

## 修正内容（How it's fixed）

### 1. ツリーシェイキング最適化と脱 `@expo/vector-icons` 実装
`node_modules` 配信ブロックをビルドプロセス（Metro側）で根本的に回避するため、アイコンのロード処理を完全に内製化しました。
   - 実際に使用している `Ionicons.ttf` と `MaterialCommunityIcons.ttf` を `assets/fonts/` へ直接手動配置。
   - `src/shared/components/icons/index.js` を新規作成し、`createIconSet` を用いて、配置したローカルのフォントファイルを直接参照する独自のアイコンラッパーを作成。
   - プロジェクト内（約25ファイル）の `import ... from '@expo/vector-icons'` を全て上記の内製アイコンラッパーへ一括置換。
   - これにより、Expo Bundler は `node_modules` の深い階層ではなく、正規の `assets/fonts/` への出力を行うようになりCloudflareの制限をネイティブに解決しました。

### 2. インフラレベルでのSPAルーティング修正
SPAのサブルートで発生する相対パスのバグを解決するため、React側での手動パス置換ハックを廃止しました。
   - `public/index.html` の `<head>` に `<base href="/" />` を追加。
   - これにより、ブラウザがURL階層に依存せず、全てのアセット（画像・フォント）を自動的にドメインのルートから解決するようになり安全かつクリーンに解決しました。

### 3. 公式ルールの明文化とスクリプトのクリーンアップ
   - 今後の事故（他の開発者やAIエージェントが誤って `@expo/vector-icons` をインポートしてしまうこと）を防ぐため、**`docs/フォント追加ルール.md`** を新規作成し、ルールを明文化しました。
   - 検証過程で作成した一時的なビルド後リネームハックスクリプトは、最終的なネイティブ解決に伴い不要となったため全て削除し、`package.json` のビルドスクリプトもクリーンアップしました。

## 新しいフォント追加時の注意事項 🚨
この変更に伴い、今後の開発ルールが一部変更となります。
詳細は **`docs/フォント追加ルール.md`** に記載していますが、要点は以下の通りです。

- **`@expo/vector-icons` からの直接インポートは今後「絶対禁止」となります。**
- 新しい種類のアイコンを使用したい場合は、必ずフォント実体（`.ttf`）を `assets/fonts/` へコピーし、`src/shared/components/icons/index.js` の内製ラッパーを経由して import する必要があります。
- このルールを破ると、本番環境において再びアイコンが全て文字化け（404エラー）する致命的なバグが再発します。
