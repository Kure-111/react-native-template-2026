# エリア場所写真の配置方法

このフォルダには、各エリアの場所を示す写真を保存します。

## ⚠️ 重要：画像を置くだけでは表示されません

React Nativeの仕様上、画像ファイルを配置するだけでは認識されません。
**必ず以下の手順を両方実行してください。**

---

## 配置手順

### 1. 画像ファイルを配置

このフォルダ（`src/features/jimu-shift/assets/`）に画像を保存します。

**対応形式：**
- `.jpg`
- `.jpeg`
- `.png`

**ファイル名：**
- エリア名と一致させてください
- 例：`ステッカー.jpeg`、`受付.png`、`本部.jpg`

---

### 2. constants.js に画像を登録（必須）

`src/features/jimu-shift/constants.js` を開き、`AREA_IMAGE_MAP` に画像を追加します。

**編集箇所：**
```javascript
export const AREA_IMAGE_MAP = {
  'ステッカー': require('./assets/ステッカー.jpeg'),
  '受付': require('./assets/受付.png'),
  '本部': require('./assets/本部.jpg'),
  // ↓ 新しい画像はここに追加
};
```

**記述例：**
```javascript
'エリア名': require('./assets/ファイル名.拡張子'),
```

---

## よくある間違い

### ❌ 画像を置いただけで終わり
```
assets/
  ├── 新しいエリア.jpg  ← 配置しただけ
```
→ **表示されません**

### ✅ 正しい方法
```
assets/
  ├── 新しいエリア.jpg  ← 配置

constants.js:
  '新しいエリア': require('./assets/新しいエリア.jpg'),  ← 登録も必要
```
→ **正しく表示されます**

---

## 画像が表示されない場合

1. `constants.js` に登録したか確認
2. エリア名とファイル名が完全に一致しているか確認（スペースや全角半角に注意）
3. ファイル拡張子が正しいか確認（.jpg, .jpeg, .png）
4. アプリを再起動してキャッシュをクリア

---

## 注意事項

- React Nativeでは `require()` が静的に解析されるため、動的な画像読み込みができません
- 画像を追加・削除した場合は、必ず `constants.js` も更新してください
- ファイル名に使える文字：日本語、英数字、ハイフン、アンダースコア
- ファイルサイズ：できるだけ軽量化してください（推奨：500KB以下）
